### üìú Hybrid Horizontal Scroll

The "About Me" section demonstrates **device-adaptive animation**:

**Desktop**: GSAP ScrollTrigger with smooth horizontal scroll  
**Mobile**: Framer Motion for native touch scrolling (no browser chrome conflicts)

```tsx
// Touch detection
useEffect(() => {
  const checkTouch = () => {
    setIsTouchDevice(
      'ontouchstart' in window ||
        navigator.maxTouchPoints > 0 ||
        navigator.msMaxTouchPoints > 0
    )
  }
  checkTouch()
}, [])

// GSAP for desktop
useGSAP(() => {
  if (isTouchDevice || !textRef.current) return

  gsap.to(textRef.current, {
    x: -(textRef.current.scrollWidth - window.innerWidth),
    ease: 'none',
    scrollTrigger: {
      trigger: containerRef.current,
      start: 'top bottom',
      end: 'bottom top',
      scrub: true,
      invalidateOnRefresh: true,
    },
  })
}, [isTouchDevice])

// Framer Motion for mobile
const { scrollYProgress } = useScroll({
  target: containerRef,
  offset: ['start end', 'end start'],
})
const x = useTransform(scrollYProgress, [0, 1], ['0%', '-50%'])
```

<Callout icon='üéØ'>
  This hybrid approach eliminates mobile browser chrome conflicts. When the
  address bar hides/shows, GSAP ScrollTrigger gets confused. Framer Motion's
  useScroll works with native scroll events, avoiding this entirely.
</Callout>

### Animation Architecture

The portfolio uses a **hybrid animation strategy**:

```tsx
// Device detection
const isMobile =
  /iPhone|iPad|iPod|Android/i.test(navigator.userAgent) ||
  'ontouchstart' in window

// ScrollTrigger configuration
ScrollTrigger.config({
  autoRefreshEvents: 'visibilitychange,DOMContentLoaded,load',
  ignoreMobileResize: true,
})

// ScrollSmoother only on desktop
if (!isMobile) {
  ScrollSmoother.create({
    smooth: 1.3,
    effects: true,
    normalizeScroll: false,
  })
}

// Cleanup on unmount
return () => {
  ScrollTrigger.getAll().forEach((trigger) => trigger.kill())
  ScrollSmoother.get()?.kill()
}
```

**Key Principles:**

1. **GSAP for precision** ‚Äì Timeline-based animations, scroll triggers, morphing
2. **Framer Motion for React** ‚Äì Component animations, gestures, springs
3. **Device detection** ‚Äì Different strategies for touch vs. non-touch
4. **Cleanup on unmount** ‚Äì Prevent memory leaks
5. **Performance monitoring** ‚Äì 60fps target with `will-change`

**Animation libraries can coexist** ‚Äì Use GSAP for timelines, Framer Motion for React components

**Cleanup is critical** ‚Äì Always dispose GSAP instances and event listeners

**Accessibility isn't optional** ‚Äì Reduced motion and keyboard nav are requirements  
**Progressive loading wins** ‚Äì Heavy assets should load after critical content
